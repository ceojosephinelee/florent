# DevLog 기록 규칙

이 프로젝트의 개발 과정 기록은 **Git이 아니라 Notion DevLog에 기록한다.**

코드는 Git이 기록하고,  
설계 의사결정과 문제 해결 과정은 **Notion DevLog**에 기록한다.

AI Agent(Codex)는 아래 상황이 발생하면 **DevLog 작성을 제안해야 한다.**

단, **자동으로 기록하기 전에 반드시 사용자에게 확인 질문을 한다.**

---

# DevLog 작성 트리거

다음 상황이 발생하면 DevLog 작성이 필요하다.

## 1. 새로운 기능 설계를 시작할 때

예

- Request 생성 기능
- 반경 검색 기능
- 알림 시스템

기록 내용

- 해결하려는 문제
- 초기 설계 아이디어
- 고려한 대안

---

## 2. 중요한 설계 결정을 했을 때

예

- 아키텍처 선택
- 기술 스택 변경
- 패턴 도입 (Outbox, Presigned URL 등)

기록 내용

- 선택한 이유
- 트레이드오프

---

## 3. AI와 중요한 설계 토론이 있었을 때

AI와의 협업 과정도 기록한다.

예

- AI가 제안한 설계
- 사용자가 수정한 부분
- 최종 선택

---

## 4. 구현 중 문제를 해결했을 때

예

- 기술적 장애 해결
- 디버깅 과정
- 성능 문제 해결

기록 내용

- 문제 상황
- 시도한 해결 방법
- 최종 해결 방법

---

## 5. 기능 구현이 완료되었을 때

예

- Request API 구현
- Radius 검색 구현
- Notification 발송 구현

기록 내용

- 구현 범위
- 예상과 달랐던 점
- 개선 포인트

---

# DevLog 작성 프로세스 (AI Agent 규칙)

AI Agent는 위 상황을 감지하면 다음 행동을 수행한다.

1. DevLog 작성이 필요한 상황인지 판단한다.

2. 작성 전에 사용자에게 질문한다.

예시 질문

"지금 DevLog를 기록할 만한 설계 결정이 발생했습니다.  
Notion DevLog에 기록할까요?"

3. 사용자가 동의하면 DevLog 초안을 작성한다.

4. 사용자가 수정 후 Notion에 저장한다.

---

# DevLog 작성 원칙

- 코드 설명은 하지 않는다
- 설계 이유를 기록한다
- 트레이드오프를 기록한다
- AI 협업 과정도 기록한다
