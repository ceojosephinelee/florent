# Prompt Logging Rule

이 프로젝트에서 사용자(개발자)가 Codex에게 입력하는 **모든 프롬프트는 Notion에 기록한다.**

이 기록의 목적은 다음과 같다.

- AI 협업 과정 기록
- 문제 해결 과정 추적
- 프롬프트 설계 개선
- 개발 회고 자료 축적

이 기록은 **코드 저장소에 남기지 않는다.**  
오직 **Notion의 "프롬프트 기록" 페이지에만 저장한다.**

---

# 기록 위치

Notion 페이지

프롬프트 기록

이 페이지는 **Database 형태로 관리한다.**

Database 구조

- Date (날짜)
- Prompts (해당 날짜의 프롬프트 목록)

각 날짜는 **하나의 페이지(row)** 로 관리한다.

---

# 기록 방식

프롬프트 기록은 다음 규칙을 따른다.

1. 사용자가 Codex에 프롬프트를 입력하면 해당 프롬프트를 기록한다.

2. 현재 날짜를 기준으로 Notion Database에서 해당 날짜의 페이지를 찾는다.

3. 만약 해당 날짜 페이지가 존재하지 않으면 새로 생성한다.

예

Date: 2026-03-07

4. 해당 날짜 페이지 안에 프롬프트를 추가한다.

프롬프트는 시간 순서대로 기록한다.

---

# 기록 포맷

각 프롬프트는 다음 형식으로 기록한다.

## HH:MM

프롬프트 내용

예

## 14:32

Spring Boot + PostgreSQL 환경에서
반경 검색 기능을 구현하려고 한다.
MVP 기준에서 가장 단순한 설계를 제안해줘.

---

## 14:47

Outbox 패턴을 MVP 알림 시스템에 적용하는 것이
과한 설계인지 분석해줘.

---

# 자동 기록 규칙

- 모든 사용자 프롬프트는 자동으로 기록한다.
- 사용자에게 기록 여부를 물어볼 필요 없다.
- 기록은 항상 Notion에만 남긴다.
- 코드 파일이나 저장소에는 기록하지 않는다.

---

# 중요 규칙

1. 프롬프트 기록은 항상 수행한다.
2. 사용자가 명시적으로 비활성화하지 않는 한 계속 유지한다.
3. 기록은 개발 흐름을 방해하지 않아야 한다.
4. 기록은 항상 Notion "프롬프트 기록" 페이지에만 저장한다.
